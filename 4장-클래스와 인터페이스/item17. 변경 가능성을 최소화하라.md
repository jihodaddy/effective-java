# Item17. 변경 가능성을 최소화하라.

## 불변 클래스
인스턴스 내부 값을 수정할 수 없는 클래스. 
String, 기본 타입의 박싱된 클래스들, BigInteger, BigDecimal   
가변클래스보다 설계하고 구현하고 사용하기 쉬움, 오류 생길 여지도 적고 안전

### 불변 클래스 만드는 규칙
- 객체의 상태를 변경하는 메서드(변경자)를 제공하지 않음.
- 클래스를 확장할 수 없도록 함.
- 모든 필드를 final로 선언.
- 모든 필드를 private로 선언.
- 자신외에 내부의 가변 컴포넌트에 접근할 수 없도록 함.
